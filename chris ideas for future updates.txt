Chris Pierce — Feature Requests Summary

Per-Plugin Transport Override — Each plugin node can toggle off the master transport and set its own BPM/time signature, enabling polyrhythms. (Simplified from his original "multiple transport groups" idea.)
Parameter Mapping System — A way to link and map parameters between plugins in the graph, similar to Bidule's parameter menu.
VST2 Support via Ildaeil — Don't host VST2 natively. Recommend users load Ildaeil (a VST3 wrapper that can load VST2 inside it). Just provide a download link.
32-bit Plugin Support via JBridge — Same approach. Don't implement natively, just link to JBridge.
Auto-Sampler — Automated tool that plays MIDI notes across 88 keys, records each note's audio output until silence is detected, and saves individual .wav files per note/velocity. Converts CPU-heavy modular chains into sample-based instruments.
Bidule Compatibility — Ensure Bidule loads and runs correctly as a VST3 plugin inside Colosseum. Treat it as a priority test case.
Color-Coded Signal Flow — Assign colors to effect chains in the node graph. Color brightness pulses in real-time based on audio magnitude. Toggle on/off to save CPU.
MIDI Output from Signal Visualization — Generate MIDI data from the color/magnitude of nodes, routable to external lighting controllers (DMX/stage lights) for live performance visual sync.
